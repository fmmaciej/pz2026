## Komendy

```bash
chmod +x plik.sh #-> dodaje uprawnienia
chmod -x plik.sh #--> usuwa uprawnienia
```

r(read) = 4, w(write) = 2, x(execute) = 1
rwx: 4 + 2 + 1 = 7

```bash 
chmod 763 plik.sh
```

7 = 4 + 2 + 1 -> rwx (właściciel)
6 = 4 + 2 -> rw (grupa)
3 -> 2 + 1 -> wx (inni)

Agrumenty "liczbowe" komendy chmod można interpretować jako zapisz liczb z systemu dziesiętnego na binarny:

`chmod 400` -> rwx rwx rwx - gdzie "1" oznacza nadane uprawnianie, a "0" - brak uprawnienia
             100 000 000 

```bash
env #-> wyświetla zmienne środowiskowe

export ZMIENNA="blabla" #(wielkie litery)
echo $ZMIENNA

history #-> lista ostatnich poleceń
history | grep git #-> (wyświetli wszystkie polecenie związane z git)
```

`!!` -> powtórzenie ostatniego polecenia

`!-2` -> powtórzenie drugiego polecenia od końca

`!158` -> odwołanie się do polecenia o numerze 158 w historii

```tr ':' '\n'``` -> zmiana : na \n

```bash
sed #-> pozwala szybko modyfikować tekst:

sed s/<przed>/<po>/<parametry>
```

Program wymieni słowo ```<przed>``` na słowo ```<po>``` i pozwala na użycie RegEx.
```<parametry>```: g - podmienia wszystkie instancje ```<przed>``` (global), i - podmienia niezależnie od case'u tekstu (case insensitive), 2 - wymieni drugą instancję ```<przed>``` (może to być dowolny numer instancji)

`~/.bashrc` - BARDZO ważny plik pozwalający na wykonywanie komend i ustawianie zmiennych globalnych przy uruchomieniu/połączeniu
