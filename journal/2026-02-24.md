## Notatki

### Działanie interfejsu i procesowania

Interfejs Miyoo napisany jest w języku `C`. Jego zadaniem jest przechwycenie flag wybranych opcji, zbudowanie na ich podstawie komendy (konkatenacja stringów) oraz wywołanie skryptu `.sh`, który przekazuje polecenia bezpośrednio do systemu.

Schemat działania:
1. Użytkownik wywołuje program przez przekazanie flag.
2. Program w `C` buduje komendę shell'ową.
3. Wywoływany jest skrypt `.sh`.
4. System wykonuje właściwe operacje.

### Docker

Docker umożliwia uruchamianie aplikacji w odizolowanych środowiskach.

#### Container
Kontener to uruchomiona instancja obrazu (`image`). Jest odizolowanym środowiskiem zawierającym aplikację i jej zależności.

#### IMG 
Obraz (`image`) zawiera system bazowy, zależności i pliki aplikacji. Budowany najczęściej z pliku `Dockerfile`, stanowi szablon dla kontenerów, których może powstać wiele na jego podstawie.

### Komenda `make`

`make` - służy do automatycznej kompilacji projektu, zgodnie ze zdefiniowanymi w `Makefile`, zasadami

```bash
make -C <ścierzka do pliku> # Przykładowe wywołanie
```

### Etapy kompilacji
- Preprocesor - przetwarzanie dyrektywy, np. #include 
- Kompilacja - przekształcenie kodu C na kod asemblera
- Asemblacja - przekształcenie kodu asemblera w kod maszynowy
- Linkowanie - łączy pliki obiektowe i biblioteki, tworząc plik wykonywalny

### Sygnatura funkcji
Określa nazwę funkcji oraz typy i liczbę jej parametrów wejściowych.

```c
int suma(int a, int b);
```

```c
int suma(int a, int b){
    return a + b;               
}
```
### Różnica między plikiem nagłówkowym a źródłowym
- plik nagłówkowy (.h) - deklaracje funkcji
- plik źródłowy (.c) - zawiera definicje funkcji
